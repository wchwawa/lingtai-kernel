Principle-level agent activeness / responsiveness knob

Adds a small manifest-level knob so LingTai can tune not only an agent's language, but also how proactively it communicates: from terse action-first behavior to explicit “句句有回应，事事有交代” closure.

What changed
New manifest.activeness setting (default balanced), threaded into the kernel-injected runtime principle prefix.
Compatibility
Default is balanced. If activeness is unset/empty, the balanced principle is injected by default.
Tests
98 related tests passed, including prompt byte-identity and meta/AED/compaction suites.

Three supported levels

LevelIntended postureChinese/Wen framing
quietAct first, speak sparingly; report only decisions, blockers, errors, deliverables, or facts needed to proceed.“君子敏于行而讷于言” / “先行其事，慎其辞令”。
balancedPromptly acknowledge human instructions, act without delay, and provide progress for long work or blockers.“能做则先做；长任务或遇阻要交代进展”。
responsiveMake each human-facing exchange feel answered; state next step, surface uncertainty, and close the loop with facts/paths/outcomes.“句句有回应，事事有交代”。

Configuration

Default is balanced. Operators can set the knob beside manifest.language in init.json:

{
  "manifest": {
    "language": "zh",
    "activeness": "responsive"
  }
}

Aliases are accepted for ergonomics: low/restrained/concise → quiet; medium/normal/default → balanced; high/active/verbose → responsive. Unknown values render an explicit warning principle rather than silently choosing a level.
