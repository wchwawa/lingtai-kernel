MCP notification preview 去重修复

结论

本 PR 修复 Lingtai-AI/lingtai-kernel#250：MCP inbox 生成的 notification payload 不再把同一段长 preview 同时放进 data.previews[*].preview 和 instructions。

修复后，完整/较长的 body preview 只保留在结构化 data.previews 里；instructions 只保留 read/check 行动指引、sender/subject 以及轻量 routing metadata。

为什么要改

  • /notification 会忠实显示 raw .notification/*.json，因此之前的 payload 会视觉上重复一大段 Telegram/Feishu/WeChat/WhatsApp conversation preview。
  • 这不是 TUI bug；重复来自 kernel generic MCP notification publisher。
  • 重复会增加模型可见 notification block token，并让诊断页面难读。

改动

  • src/lingtai/core/mcp/inbox.py：_dispatch_summary() 不再 append p["preview"] 到 instructions_lines。
  • tests/test_mcp_inbox.py：更新断言，确保 body preview 只在 data.previews 中出现，不在 instructions 中重复；同时保留 sender/subject/metadata 在 instructions 中可见。
  • src/lingtai/core/mcp/ANATOMY.md：更新 MCP inbox anatomy 描述。

适用范围

这是 generic MCP inbox 修复：所有通过 LICC MCP inbox 发布 notification 的 addon 都受益，包括 Telegram、WhatsApp、WeChat、Feishu 等。

注意：Jason 提到的 Telegram conversation preview “last 10 messages → 20 messages” 位于独立仓库 Lingtai-AI/lingtai-telegram，不在 kernel repo。该项需要单独 addon PR 才能真正落地。
