Conclusion

This is a minor 0.x release rather than another 0.12 patch because the window contains agent/tool API compatibility changes: notification operations move to a mandatory standalone notification capability, and agent-initiated molting now requires a durable session_journal_path.

Highlights

  • Progressive disclosure and result hygiene: system.summarize, constant _tool_result_metadata, dismissible large-result reminders, and clearer top-level prompt guidance.
  • Notification architecture: actual notification-block snapshots are persisted; notification verbs are split out of system into a dedicated notification tool with atomic dismiss semantics.
  • Memory safety: agent-initiated molting now validates a non-empty session journal entry before context is shed.
  • Codex reliability: cache-affinity rotation/refresh behavior is hardened; the runtime now sends underscore session_id/thread_id headers and honest LingTai client-identity metadata.
  • Daemon observability: terminal-state notifications and claude-p daemon token usage are surfaced.
  • i18n cleanup: internal operating/tool-schema text falls back to English while operator-visible strings remain localized.

Compatibility notes

  • Use the standalone notification capability for check/dismiss operations; old system.notification / system.dismiss aliases are gone.
  • psyche.context.molt now requires session_journal_path for deliberate agent molting.
  • Public install/upgrade messaging should continue to describe the TUI-created project venv as the normal user path; PyPI is the kernel package source and developer/diagnostic validation target.
