PR #257 dynamic language principle prompt fix

lingtai-kernel system prompt language constraint

Problem

The kernel accepted an agent language value when building the system prompt, but the prompt builder did not use it as a hard, model-visible constraint. Recipes and static prompt assets could therefore pull an English agent toward nearby Chinese source text.

Change

  • Adds a kernel-injected dynamic language principle at the very start of the system prompt.
  • Supports en, zh, and wen with the constraint written in the target language.
  • Unknown language codes fall back to English wording while preserving the raw code.
  • The constraint says ordinary prose and human-facing replies must use the configured language unless a human/task explicitly asks otherwise; quoting source text, code, and identifiers is allowed.
  • Both string and batched prompt builders include the same first principle and remain byte-identical when batches are joined with the normal fallback separator.

Files

  • src/lingtai_kernel/prompt.py — dynamic principle table/helper and builder integration.
  • tests/test_prompt.py — ordering, language variants, fallback, and batch/string byte-identity tests.
  • src/lingtai_kernel/base_agent/prompt.py, src/lingtai_kernel/ANATOMY.md — documentation/anatomy updates.
